# macOS 内置盘空间治理

## 一句话的道

> 内置盘是热工作台，不是仓库；外置盘 / SMB 是冷库，不是活体系统盘。磁盘治理的本质，是让数据按热度、风险、可恢复性有秩序地流动。

## 先守住边界

- 不把“清理”当成目标；目标是建立长期秩序。
- 不在没有确认时删除、迁移、定时执行变更、创建软链。
- 不把 app 数据库、云同步控制目录、系统关键目录当普通文件夹处理。
- 默认先只读审计和 dry-run；真正变更前必须说明会发生什么、如何验证、如何回滚。

## 三层术

### 1. 先看见：定时提醒检查

适合所有用户。只做报告，不自动移动文件。

- 先跑 `scripts/triage.sh` 或 `ncdu -x` 建立基线。
- 设定阈值，例如内置盘低于 60 GB、Downloads 超过 20 GB。
- 建议周 / 月 / 季度提醒；提醒内容应该是“检查并建议”，不是“自动删除”。
- 如果宿主环境支持提醒或自动化，可在用户确认后创建；否则给出日历、提醒事项或 launchd 方案。

### 2. 再流动：不可软链的周期归档

适合 Downloads、会议录像、导出目录、扫描件、工程交付物等“源目录必须继续是真目录”的场景。

- 原目录继续留在内置盘，新文件仍正常落入本地。
- 按年龄阈值把冷文件归档到 `/Volumes/...`。
- 先用 `scripts/periodic_archive.sh` dry-run；确认后再 `--apply`。
- 只有在归档已验证且目标盘有备份时，才考虑 `--remove-source-files`。

### 3. 最后重构：可软链的永久迁移

适合用户可控、低频、读多写少、路径又需要保持不变的大目录。

- 迁移前确认外置盘 / SMB 足够可靠，且目标未挂载时用户能接受路径失效。
- 用 `scripts/relocate_with_symlink.sh` 先复制，再把原目录改名为备份，最后创建软链。
- 保留时间戳备份，至少经历一次重启 / 重新挂载验证后再考虑删除。
- 不建议软链 Photos、Mail、Messages、iCloud/Dropbox/OneDrive/Google Drive 根目录、Docker 磁盘镜像、VM、数据库、活跃仓库。

## 执行顺序

1. **识别人群与目标**
   - 办公室白领：用人能理解的分类，优先 Downloads、桌面、文档、会议录像、导出文件。
   - 技术工程师：加入缓存、构建产物、包管理器、模拟器、Docker、模型、数据集、仓库产物。

2. **检查工具与目标盘**
   - 缺工具时优先用 `brew install ncdu rsync`。
   - 外置盘或 SMB 必须已挂载在 `/Volumes/...`。
   - 不要把未挂载路径当目标；SMB 中断时应能重跑 `rsync` 修复。

3. **只读审计**
   - 先跑 `scripts/triage.sh --persona office|engineer|mixed --target "$HOME"`。
   - 需要交互查看时用 `ncdu -x "$HOME"`，`-x` 避免扫进外置盘或网络盘。
   - 系统级扫描可能需要 Full Disk Access 或 `sudo`，先说明权限影响。

4. **分类决策**
   - **保留本地：** 活跃文档、app 数据库、钥匙串、云同步控制目录、活跃仓库。
   - **可删除 / 可重建：** 缓存、构建产物、包缓存、旧模拟器、重复生成物。
   - **一次性归档：** 旧下载、旧导出、安装包、历史项目快照。
   - **周期归档：** 持续增长的录像、导出、扫描、下载目录。
   - **软链迁移：** 大模型、数据集、只读素材库等安全目录。

5. **确认后执行**
   - 一次性归档：`scripts/rsync_archive.sh`。
   - 周期归档：`scripts/periodic_archive.sh`。
   - 软链迁移：`scripts/relocate_with_symlink.sh`。
   - 删除、移除源文件、创建定时任务、创建软链都必须单独确认。

## 资源

- `references/long-term-governance.md`：道与术、节奏、阈值、三类治理模式。涉及长期提醒、周期迁移、软链迁移时先读它。
- `references/playbooks.md`：办公室白领和技术工程师的具体路径、禁区、报告模板。
- `scripts/triage.sh`：只读盘点内置盘、挂载盘、常见候选目录。
- `scripts/rsync_archive.sh`：一次性归档，默认 dry-run，不删除源文件。
- `scripts/periodic_archive.sh`：不可软链目录的周期冷文件归档，默认 dry-run。
- `scripts/relocate_with_symlink.sh`：可软链目录的永久迁移，默认 dry-run，保留回滚备份。

## 输出格式

给用户的方案应包含：

- 当前压力：内置盘剩余空间、主要增长点。
- 治理模式：提醒检查 / 周期归档 / 软链迁移 / 删除重建。
- 节奏阈值：多久检查一次、多少天归档、低于多少空间触发行动。
- 行动表：路径、类型、预计释放、命令或 app 操作、风险。
- 确认门：哪些步骤会复制、移动、删除、定时执行或创建软链。
- 验证与回滚：如何打开样本文件、确认软链、恢复备份。
